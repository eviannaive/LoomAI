請協助我規劃一個 Flutter 專案，用於開發一款名為「LoomAI」的 AI 聊天 App。

**專案核心目標：**
1.  **應用名稱**：LoomAI
2.  **平台**：iOS & Android (最終目標上架至 App Store 和 Google Play)
3.  **核心功能**：讓使用者能夠創建、編輯並與個性化的 AI 角色進行互動對話，並具備記憶管理、回憶儲存與訂閱機制。
4.  **UI/UX 風格**：溫柔、夢幻，並略帶 cyberpunk 風格（請協助產出 UI Mockup 建議）。

**App 頁面與導覽規劃：**
App 將包含底部導覽列 (Bottom Navigation Bar)，提供以下四個主要頁面：
* **對話 (Chat)**：AI 角色互動的核心聊天介面。
* **記憶碎片 (Character Hub)**：管理與選擇已創建 AI 角色，每個角色卡片上應有一個「啟動碎片」按鈕以開始對話。
* **回憶盒子 (Memory Box)**：儲存用戶喜歡的聊天記錄或特定對話片段。
* **檔案 (Profile)**：顯示用戶資料、訂閱狀態和相關設定。

**核心功能詳述：**

1.  **AI 角色創建與編輯：**
    * **基礎資訊**：姓名、頭像、背景故事。
    * **個性設定**：詳細設定角色的性格特徵與說話風格（此將直接用於 OpenAI 的系統提示 `system message`）。
    * **擬人化偏好**：設定角色喜歡什麼、不喜歡什麼，以豐富角色表現。

2.  **對話系統：**
    * **模型選擇**：支援 GPT-4 和 GPT-3.5 模型切換。
    * **參數設定**：允許用戶調整 OpenAI API 的 `temperature`、`top_p` 等參數。
    * **訊息顯示**：聊天訊息應支援 Markdown 格式（粗體、斜體、列表等）和表情符號顯示。

**商業邏輯與限制：**

* **免費額度**：新用戶提供 10,000 字的免費對話額度。
* **訂閱制度**：超過免費額度後，需訂閱才能繼續使用。月費設定為 NT$300，訂閱後無字數限制。

**記憶管理機制 (關鍵設計)：**

* **長期記憶保留**：用戶與每個 AI 角色的互動（如戀愛式對話、經歷過的事件）需被長期保留，確保對話的連貫性和深度。
* **記憶呈現**：「記憶碎片」頁面應以時間軸或事件軌跡的形式展示與特定角色的互動歷史。
* **記憶儲存方式**：建議採用記憶摘要 (memory summarization) 或 embedding 方式來管理和存取長期記憶。
* **角色專屬記憶**：每個 AI 角色都應有獨立的記憶檔案，這些檔案可以定期彙總，或作為後續對話的上下文 (`context`) 輸入。

**技術堆疊建議：**

* **前端框架**：Flutter。
* **狀態管理**：推薦使用 Provider、Riverpod 或 Bloc 進行狀態管理。
* **AI API 串接**：使用 OpenAI API，主要調用 `chat/completions` 端點。
* **資料儲存**：**初期規劃不串接外部資料庫**，建議先使用本地儲存方案（如 `shared_preferences` 或 `hive`）來處理用戶數據、角色設定和聊天記錄，特別是記憶管理部分。
* **程式碼品質**：請考慮專案結構的清晰度、可擴展性和維護性。

**最終成果要求：**

* 提供 Flutter 專案的初始程式碼結構。
* 包含主要頁面的導覽和基本 UI 框架。
* 針對 UI 風格（溫柔、夢幻、略帶 cyberpunk）提供詳細的設計指南或 Mockup 建議，包含色彩方案、字體、按鈕樣式等。
* **特別強調記憶管理機制的設計思路和初步實作方向**，包括如何處理長期對話記憶的存儲與檢索。
* 規劃好 OpenAI API 串接的基礎架構和模型/參數切換邏輯。
* **不需實作完整的商業訂閱邏輯，但需預留擴展接口。**
